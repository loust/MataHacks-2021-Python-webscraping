It depends on the type of editing.

For creating and altering tables I use CLSQL, bindings to sql for common lisp ( slackware comes with clisp in the D set ). But for using the database (inserting, changing, or removing records), I think the database client that comes with openoffice.org provides a sufficient gui, once you set it up.

It is also not too difficult to create cgi scripts to access mysql database from common html forms, and design graphical front end using standard web design techniques. The cgi scripts can be written in almost anything as most everything has bindings to mysql; thus you can use C, C++, perl, php, python, ruby, or even clisp to create a cgi script to interact with your database on behalf of the input gathered by an html form. Of course this has been done many times already, so you can just use what's already been done, such as phpmyadmin... but you could also deploy some qt3 or qt4 widgits to gather the info if you don't want to use a web browser, or gtk widgits, or tk widgits, etc.

Some time ago I used NetBeans. It wasn't overly powerful it terms of functionality, but it did what it did flawlessly.

Believe it or not, someday I started to like graphical frontends with visualisation capabilities for the database structure, and ended up using MS Access and OpenOffice.org Base.
Once I found that there are serious problem with foreign key "inheritance" in MS Access I started to double check everything with additional tools, mostly database system specific.

A quick web search lead me to these alternatives, you might want to look at:
easyDBase
DBVisualizer
Okapi
Jailer

These are all tools supporting more than one database product. You should easily find more specialised (optimised?) tools, if you are interested in support for just one particular product. Especially for MySQL there must be tons of tools, and certainly some good ones, too...

Goold luck!

Try Vim!
I refrained from responding to this thread early because you were looking for an "SQL editor", so did not want to seem flippant about it. But...

My main work these days is relational db design and support, mostly MySQL, PostgreSQL and DB2, and after many frustrations looking for an "easy" front end I have found that good old Vi/Vim turns out to be the most productive for me - by far!

I use a graphical ER tool for schema design mostly because of the documentation it produces, and I model a lot with UML, but for efficient query development you can't beat the combination of Vi/Vim (or other plain text editor), the native DB shell client and a few simple, good habits! Try it.

I also use phpmyadmin for many related chores at times, but for interactive, ad hoc queries open a shell and the native client and dive in. For all but trivial queries write them in Vi/Vim, execute in client, rinse and repeat... save for later use.
